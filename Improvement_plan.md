# 改善計画メモ

## 概要
- 既存の解析不備（正解抽出・例外処理）は前回までに改善した。
- 今回、新たに UI 表示面で以下の問題が確認された。
  1. 長い選択肢文がラジオボタン内で折り返されず、右端で見切れる。
  2. 表形式の問題が表として表示されず、ヘッダのみ本文に残って行データが選択肢へ崩れている。
- 実データ確認の結果、表形式の問題は少なくとも `2Q.md` の No.50 と `4Q.md` の No.45 に存在する。

## 確認した問題点

### 1. 長い選択肢の見切れ
- 対象: `src/quiz_app/app.py`
- 症状:
  - ラジオボタンの選択肢テキストが横幅いっぱいで切れ、全文を読めない。
- 原因:
  - Streamlit 標準のラジオ表示に折り返し調整がなく、長文時の視認性が不足している。
- 影響:
  - 特に法規問題の長い記述で可読性が大きく低下する。

### 2. 表形式問題の表示崩れ
- 対象: `src/quiz_app/app.py`
- 症状:
  - 問題本文には表ヘッダだけが表示され、表の各行は `radio` の選択肢文字列として崩れて表示される。
- 再現確認済み:
  - `md/2Q.md` No.50
  - `md/4Q.md` No.45
- 原因:
  - `parser.py` は表の各行を選択肢として正常に抽出しているが、UI 側が表形式問題を通常の選択肢問題として描画している。

## 改善方針

### 方針A（採用）
- `app.py`
  - ラジオボタンの選択肢テキストに折り返し CSS を適用する。
  - 表形式問題を検出し、本文テキストと表データを分けて描画する。
  - 表形式問題では、表は Markdown テーブルとして表示し、ラジオの選択肢ラベルは短い形式（例: `選択肢 1`）にする。
- `tests/`
  - 表形式問題の表示用ヘルパーのテストを追加する。
  - 長文選択肢の折り返し CSS 文字列のテストを追加する。
  - 既存 UI テストを壊さず、追加の表示テストで回帰を防ぐ。

### 方針B（今回は不採用）
- `parser.py` に表形式専用の構造化データモデルを追加する。
- 将来的には有効だが、今回は UI 修正だけで解決可能であり差分が大きい。

## 実装タスク
1. 表形式問題の表示仕様をテストで固定する。
2. 長文選択肢の折り返し仕様をテストで固定する。
3. UI 側に表形式問題レンダリングヘルパーを追加する。
4. UI 側にラジオ選択肢の折り返し CSS を追加する。
5. `uv run ruff check src/ tests/` と `uv run pytest` で検証する。
