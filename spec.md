# 仕様・設計書 (Specification)

## 1. アーキテクチャ設計（関心の分離）
本システムは、UIの都合にロジックが引きずられないよう、以下の3層アーキテクチャを採用する。

### 1.1. Data Layer: `parser.py` (解析モジュール)
* **役割:** Markdownファイルの読み込みと構造化データへの変換。Streamlitには一切依存しない。
* **主要データ構造 (DataClassを使用):**
    * `Choice`: 選択肢のテキストと番号
    * `Question`: 問題番号、問題文テキスト、`Choice`のリスト（要素数は可変）、正解番号、解説テキスト
* **振る舞い:** QファイルとAファイルを照合し、`Question`オブジェクトのリストを生成する。ファイルが存在しない場合やフォーマット不正の場合は適切なカスタム例外を送出する。

### 1.2. Logic Layer: `engine.py` (クイズエンジンモジュール)
* **役割:** クイズの進行状態（State）の管理。UIへの表示方法は関知しない。
* **主要クラス:** `QuizEngine`
* **保持する状態:**
    * 読み込まれた全問題リスト
    * 出題対象の問題リスト（ランダム化・件数絞り込み適用後）
    * 現在の問題インデックス
    * ユーザーの解答履歴とスコア
* **主要メソッド:**
    * `start_quiz(questions, is_random, limit)`: クイズを初期化する。
    * `get_current_question()`: 現在の問題を返す。
    * `submit_answer(choice_number)`: 解答を記録し、正誤を返す。
    * `get_results()`: スコアと間違えた問題のリストを返す。

### 1.3. UI Layer: `app.py` (Streamlitフロントエンド)
* **役割:** ユーザーインターフェースの描画と入力の受け付け。`parser.py`と`engine.py`を呼び出す。
* **状態管理:** Streamlitの `st.session_state` を使用して `QuizEngine` のインスタンスを保持し、画面遷移（設定画面 → プレイ画面 → リザルト画面）を管理する。
* **ダークモード:** Streamlitのテーマ設定（`.streamlit/config.toml` またはシステム設定依存）を利用し、視認性の高いUIを構築する。

## 2. テスト方針 (TDD)
* まず `parser.py` と `engine.py` に対する `pytest` を記述する。
* UI（`app.py`）を起動しなくても、コアロジックが100%動作することをテストで証明してからUIの実装に移る。
