# 要件定義書 (Requirements)

## 1. プロジェクト概要
マークダウン形式の選択問題（Q）と解答・解説（A）を読み込み、ブラウザ上で学習できるインタラクティブなWebベースのクイズアプリケーション。

## 2. 目的と対象ユーザー
* **目的:** 過去問・模試テキストのデジタル化と効率的な反復学習。
* **対象ユーザー:** 資格試験の学習者。

## 3. システム要件（非機能要件）
* **開発言語:** Python
* **パッケージ管理:** `uv`
* **実行環境:** Podmanコンテナ
* **UIフレームワーク:** Streamlit
* **アーキテクチャ:** 関心の分離（UI層、ロジック・状態管理層、データ解析層の厳格な分割）
* **品質保証:** `pytest` によるTDD、`Ruff` による静的解析・フォーマット

## 4. データソースの仕様
* **問題ファイル（Q）と解答ファイル（A）:**
    * 「### Ｎo. X」の見出しで問題が区切られている。
    * **[重要] 選択肢の数は固定ではなく、問題によって変動する（例: 4択、5択など）。プログラムは動的にこれに対応する。**
    * 解答ファイルには正解番号と解説テキストが含まれる。

## 5. 主要機能要件（ユーザー体験）
1. **出題設定:** ファイル選択、出題順（順次/ランダム）、出題数の指定。
2. **クイズ実行:** 問題表示、可変選択肢からの解答入力、即時正誤判定、解説表示。
3. **リザルト:** 最終スコア表示、間違えた問題の復習機能。
4. **UI/UX:** **ダークモード対応**（Streamlitの標準機能を活用、またはCSSで明示的にサポート）。
