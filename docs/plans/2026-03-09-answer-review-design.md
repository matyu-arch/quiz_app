# 解答後レビュー表示 設計メモ

## 目的
- 解答後画面で、元の問題文が見えにくい/見えない問題を解消する。
- ユーザーが「何を選んだか」「何が正解か」を即座に把握できるようにする。

## 現状の課題
- 解答後画面では、問題文は描画している実装だが、実際の利用画面では解説の情報量が多く、ユーザー視点では問題文や選択肢との対応関係が追いにくい。
- 選択肢ごとの結果表示がなく、正解/不正解だけでは復習効率が低い。

## 方針
- 解答後画面で、問題文・表・選択肢一覧を明示的に再表示する。
- 選択肢一覧には以下のマークを付ける。
  - 正解した場合: 選んだ選択肢のみに `✅`
  - 不正解した場合: 選んだ選択肢に `❌`、正解の選択肢に `✅`
- 表形式問題でも同じレビュー表示を適用する。

## UI構成
1. 問題番号
2. 元の問題文
3. 表（ある場合）
4. 正誤の強調ボックス
5. マーク付き選択肢一覧
6. 解説
7. 次の問題へ

## 実装方針
- `app.py` にレビュー表示用ヘルパーを追加する。
- 選択結果を保持するため、`session_state` に最後に選んだ選択肢番号を保存する。
- テストは `tests/test_app.py` に追加し、通常問題と表形式問題の両方を確認する。
